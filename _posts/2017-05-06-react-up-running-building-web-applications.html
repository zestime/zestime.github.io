Why this

팀내에서 React를 사용하면서, 조금 더 심도 있는 접근을 위해서 React라는 주제로 책을 선정하였습니다. 사실, 구글링과 스택오버플로우로 얼룩진 개발 시간을 구제하기 위해서 기초를 쌓으려고 읽었습니다. React도 이제 생각보다 연식이 있는지라, 가장 최근의 서적으로 Facebook 개발자가 작성한 책입니다.

How's

전체적으로 초심자를 고려하여 작성된 책입니다. 즉, React 초짜에게 정말 좋은 길라잡이가 되어줄 수 있을 것 같습니다. 팀에서는 어느 정도 React에 익숙해진 단계였기 때문에, 가볍게 여기시는 분들도 계셨습니다. 하지만, 예제라던지 다소 Concrete하지 않은 개념들을 잡아주기에는 좋은 실용서로 생각됩니다. 책은 React의 기초에서 시작해서, React 개발 전반을 다루고 있습니다. Flux와 같은 Store에 대한 개념도 중립적으로 기술하고 있습니다. 좀 더 효과적인 개발을 위해 고민하시는 분들에게 도움이 될 수 있도록, 개발 환경에 적용하기 좋은 주제들(Testing, Linting, Flow)도 다루고 있습니다. (사실, Flow에 대한 도입을 팀원들에게 약속했으나, 아직도 적용되지 않았네요;;;)

Pros)
- 아무리 강조해도 지나치지 않을 기초
- 간결하게 중요한 것만 다루는 스타일
- Excel 예제를 발전시켜 나가는 구조로, 착안점이나 배울 점이 많음
- React를 둘러싼 다른 library로 소화

Cons)
- React에 내부적인 이야기를 기대했다면 아숩
- 책의 앞 챕터는 babel을 다루지 않고 설명하는데, 다소 지루했음
